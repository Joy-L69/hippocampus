# Hippocampus 🧠

**一个拥有人脑式分级记忆的 AI 智能体**

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

## 💡 核心理念

当前的 AI 智能体（Agent）在执行多步任务时，普遍采用“全量回顾”的方式——每次推理都把所有历史记录塞进上下文。这导致：

-   **词元（Token）消耗巨大**：90% 的推理成本花在了反复回忆上
-   **智能退化**：长上下文中的“迷失中间”现象，让 Agent 逐渐忘记最初的目标

**Hippocampus 提出了一个不同的答案：像人脑一样记忆。**

人脑不会在每次思考时回顾一生，而是：
1.  **优雅地遗忘**：不重要的细节自动沉底
2.  **按需检索**：只有当需要时，才去提取相关记忆
3.  **分级存储**：短期工作记忆保持极简，长期记忆静待索引

本项目就是这套理念的一个可运行原型。

## 🎯 核心功能

-   **总结并记忆**：将重要对话提炼成摘要，存入本地向量数据库
-   **语义检索回忆**：根据当前问题，精准检索出最相关的历史记忆
-   **按需注入上下文**：只把检索到的记忆片段作为背景，避免全量回顾
-   **零 Token 检索**：记忆检索过程完全在本地完成，不消耗任何 API 费用
-   **完全本地化**：你的记忆数据只在你自己的硬盘上，不上传任何云端

## 🧰 技术栈

-   **大模型 API**：DeepSeek Chat / OpenAI GPT-4o-mini（可自行切换）
-   **向量数据库**：ChromaDB（本地部署，首次运行会自动下载嵌入模型）
-   **用户界面**：Streamlit
-   **语言**：Python

## 🚀 快速开始

### 前提条件

-   Python 3.9+
-   一个 DeepSeek 或 OpenAI 的 API Key

### 安装与运行

```bash
# 1. 克隆本仓库
git clone https://github.com/你的用户名/hippocampus.git
cd hippocampus

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动应用
streamlit run app.py


使用说明

在浏览器打开 http://localhost:8501

在侧边栏填写你的 API Key

存入记忆：勾选“总结并记忆”，然后发送消息

检索记忆：输入问题后，点击“搜索相关记忆”，再发送

观察：刷新页面后，直接提问（失忆） vs 点击搜索后提问（精准回忆）

🤔 常见问题

Q：为什么第一次存入记忆时那么慢？
A：ChromaDB 首次运行会自动下载一个嵌入模型（约 80MB）。这是本地检索的必要组件，只需下载一次，后续存入和检索都会很快。

Q：记忆存在哪里？
A：默认存储在系统的 Chroma 缓存目录（如 C:\Users\你的用户名\.cache\chroma\）。你也可以修改 app.py 中的路径，将其放在项目文件夹内。

Q：如何更换模型？
A：在 app.py 中找到 model= 和 base_url= 的配置，修改为你想要的大模型即可。
